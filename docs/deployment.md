# Deployment Guide

This document covers deployment configurations for World In a Pie (WIP) across different environments.

---

## Quick Start

The recommended way to deploy WIP is using the unified setup script:

```bash
# Auto-detect platform and use sensible defaults
./scripts/setup.sh

# Access: https://localhost:8443 (accept self-signed cert)
# Login: admin@wip.local / admin123 (Dex OIDC)
# Or API Key: dev_master_key_for_testing
```

## Deployment Profiles

| Profile | Target | Auth | Mongo Express | RAM Usage | Description |
|---------|--------|------|---------------|-----------|-------------|
| **mac** | Mac development | Dex + API keys | Yes | ~1.5GB | Full stack for development |
| **pi-minimal** | Pi with limited RAM | API keys only | No | ~800MB | No OIDC, minimal services |
| **pi-standard** | Pi 4 (2-4GB) | Dex + API keys | No | ~1GB | Full stack without Mongo Express |
| **pi-large** | Pi 5 (8GB+) | Dex + API keys | Yes | ~1.2GB | Full stack with Mongo Express |
| **dev-minimal** | Any platform | API keys only | Yes | ~1.2GB | Quick testing without OIDC |

All profiles use:
- **MongoDB** for document storage
- **PostgreSQL** for reporting
- **NATS with JetStream** for messaging
- **Caddy** for reverse proxy (HTTPS) - except minimal profiles

---

## Network Modes

The setup script supports three network modes:

| Mode | Description | Use Case |
|------|-------------|----------|
| `localhost` | Only accessible from local machine | Mac development (default) |
| `remote` | Only accessible from network | Headless Pi |
| `both` | Both localhost and network access | Pi accessible from anywhere (default for Pi) |

```bash
# Mac localhost (default)
./scripts/setup.sh

# Mac with network access for mobile testing
./scripts/setup.sh --network both --hostname dev-mac.local

# Pi standard with OIDC
./scripts/setup.sh --profile pi-standard --hostname wip-pi.local

# Pi minimal for API-only deployments
./scripts/setup.sh --profile pi-minimal
```

## Manual Development Setup

If you prefer manual control over services:

```bash
# 1. Start shared infrastructure
podman-compose -f docker-compose.infra.yml up -d

# 2. Start Registry and initialize namespaces
cd components/registry
podman-compose -f docker-compose.dev.yml up -d

curl -X POST http://localhost:8001/api/registry/namespaces/initialize-wip \
  -H "X-API-Key: dev_master_key_for_testing"

# 3. Start remaining services
cd ../def-store && podman-compose -f docker-compose.dev.yml up -d
cd ../template-store && podman-compose -f docker-compose.dev.yml up -d
cd ../document-store && podman-compose -f docker-compose.dev.yml up -d
cd ../reporting-sync && podman-compose -f docker-compose.dev.yml up -d

# 4. Start WIP Console
cd ../../ui/wip-console
podman-compose -f docker-compose.dev.yml up -d

# Or run Console locally for development
npm install && npm run dev
```

## Storage Configuration

Data is stored in `./data/` by default. Override with `--data-dir` or `WIP_DATA_DIR`:

```bash
# Use external storage (USB SSD on Pi, NFS share, etc.)
./scripts/setup.sh --data-dir /mnt/usb-ssd --profile pi-standard --hostname wip-pi.local

# Or via environment variable
WIP_DATA_DIR=/mnt/usb-ssd ./scripts/setup.sh
```

### Storage Subdirectories

| Directory | Contents |
|-----------|----------|
| `mongodb/` | Document store data |
| `postgres/` | Reporting database |
| `nats/` | Message queue persistence |
| `dex/` | OIDC tokens and state |
| `caddy/data/` | TLS certificates |
| `caddy/config/` | Caddy configuration state |

See [storage.md](storage.md) for detailed storage options (USB SSD, NFS, external drives).

## Raspberry Pi Deployment

### Prerequisites

```bash
# Install Podman on Raspberry Pi OS
sudo apt update
sudo apt install -y podman podman-compose jq

# Clone repository
git clone http://192.168.1.17:3000/peter/World-In-A-Pie.git
cd WorldInPie
```

### Quick Setup

```bash
# Auto-detect Pi model and deploy
./scripts/setup.sh --hostname your-pi.local

# Or specify profile explicitly
./scripts/setup.sh --profile pi-standard --hostname your-pi.local
./scripts/setup.sh --profile pi-minimal  # API keys only
./scripts/setup.sh --profile pi-large --hostname your-pi.local  # Pi 5 8GB+

# With external storage
./scripts/setup.sh --data-dir /mnt/usb-ssd --profile pi-standard --hostname your-pi.local
```

### Why Caddy for OIDC?

The OIDC library (oidc-client-ts) uses PKCE which requires `Crypto.subtle`, available only in secure contexts (HTTPS or localhost). Caddy provides:

- Auto-generated self-signed TLS certificate
- Reverse proxy for all services on single port (443)
- OIDC login works over network without SSH tunnels
- ~25MB RAM overhead

For API-key-only deployments, use `--profile pi-minimal` to skip Caddy.

### Pi-Specific Notes

| Consideration | Details |
|---------------|---------|
| **MongoDB version** | Pi 4 requires MongoDB 4.4.x (ARMv8.0), Pi 5 can use 7.x |
| **Health check timing** | Pi profiles use longer intervals (10s vs 5s) |
| **Mongo Express** | Excluded from pi-standard to save ~200MB RAM |
| **SD Card wear** | Consider external USB SSD for data directory |

## Backup and Recovery

### MongoDB Backup

```bash
#!/bin/bash
# backup-mongodb.sh

BACKUP_DIR=/backups/mongodb
DATE=$(date +%Y%m%d_%H%M%S)

# Dump database
podman exec wip-mongodb mongodump --out="/data/backup_$DATE"

# Copy from container
podman cp wip-mongodb:/data/backup_$DATE "$BACKUP_DIR/$DATE"

# Compress
tar -czf "$BACKUP_DIR/wip_$DATE.tar.gz" -C "$BACKUP_DIR" "$DATE"
rm -rf "$BACKUP_DIR/$DATE"

# Retain last 30 days
find "$BACKUP_DIR" -name "*.tar.gz" -mtime +30 -delete

echo "Backup complete: $BACKUP_DIR/wip_$DATE.tar.gz"
```

### PostgreSQL Backup

```bash
#!/bin/bash
# backup-postgres.sh

BACKUP_DIR=/backups/postgres
DATE=$(date +%Y%m%d_%H%M%S)

# Dump database
podman exec wip-postgres pg_dump -U wip wip_reporting | gzip > "$BACKUP_DIR/wip_reporting_$DATE.sql.gz"

# Retain last 30 days
find "$BACKUP_DIR" -name "*.sql.gz" -mtime +30 -delete

echo "Backup complete: $BACKUP_DIR/wip_reporting_$DATE.sql.gz"
```

### Recovery

```bash
# MongoDB restore
tar -xzf wip_20240115_120000.tar.gz
podman cp 20240115_120000 wip-mongodb:/data/
podman exec wip-mongodb mongorestore /data/20240115_120000

# PostgreSQL restore
gunzip -c wip_reporting_20240115_120000.sql.gz | \
  podman exec -i wip-postgres psql -U wip wip_reporting
```

## Monitoring

### Health Endpoints

Each service exposes health endpoints:

```
GET /health    - Basic liveness check
```

### Reporting-Sync Metrics

The reporting-sync service provides comprehensive monitoring:

```
GET /metrics           - Latency percentiles, throughput, per-template stats
GET /metrics/consumer  - NATS consumer info (queue depth, pending)
GET /alerts            - Active alerts and configuration
```

### Key Metrics (Reporting-Sync)

| Metric | Description |
|--------|-------------|
| `sync_latency_p50/p95/p99` | Sync latency percentiles |
| `events_processed_total` | Total events processed |
| `events_per_second` | Current throughput |
| `queue_pending` | NATS pending messages |
| `template_stats` | Per-template sync statistics |

### Alert Thresholds

Configure alerts via `PUT /alerts/config`:

```json
{
  "queue_lag_threshold": 100,
  "error_rate_threshold": 0.05,
  "stalled_threshold_seconds": 300,
  "webhook_url": "https://your-webhook.example.com"
}
```

## Troubleshooting

### Common Issues

| Issue | Cause | Solution |
|-------|-------|----------|
| API won't start | MongoDB not ready | Check `podman logs wip-mongodb` |
| Auth failures | Dex misconfigured | Verify config/dex/config.yaml issuer URL |
| OIDC login fails | Wrong network mode | Ensure issuer URL matches access URL |
| Sync lag | Queue backlog | Check `curl localhost:8005/metrics/consumer` |
| Out of memory | Wrong profile | Use pi-minimal or pi-standard for limited RAM |
| Self-signed cert warning | Expected behavior | Click 'Advanced' -> 'Proceed' |

### Debug Mode

```bash
# Run setup with debug output
./scripts/setup.sh --debug

# View service logs
podman logs -f wip-registry
podman logs -f wip-document-store
podman logs -f wip-reporting-sync
```

### Container Management

```bash
# View all WIP containers
podman ps --filter "name=wip-"

# Stop all services
podman stop $(podman ps -q --filter "name=wip-")

# Remove all containers and volumes
podman rm -f $(podman ps -a -q --filter "name=wip-")
podman volume rm $(podman volume ls -q --filter "name=wip-")

# Reset and restart
./scripts/setup.sh
```

### Direct Database Access

```bash
# MongoDB shell
podman exec -it wip-mongodb mongosh

# PostgreSQL shell
podman exec -it wip-postgres psql -U wip -d wip_reporting

# NATS monitoring
curl http://localhost:8222/jsz  # JetStream info
```
