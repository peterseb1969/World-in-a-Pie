# App Containerization Guide

How to package a WIP app for production deployment alongside WIP core services.

This guide distills lessons from deploying React Console and ClinTrial Explorer on a Raspberry Pi (2026-04-10). Every gotcha listed here was discovered the hard way.

---

## The App Contract

An app ships **one file**: `docker-compose.app.<name>.yml`. This is a compose chunk that `setup-wip.sh` discovers and integrates into the WIP deployment.

```yaml
services:
  <name>:
    image: <registry>/<name>:<tag>
    container_name: wip-<name>
    labels:
      wip.app: "true"
      wip.app.name: "Human-Readable Name"
      wip.app.route: "/apps/<slug>"
      wip.app.port: "<internal port>"
      # Optional — triggers Dex OIDC client generation:
      wip.app.oidc: "true"
      wip.app.oidc.client_id: "<client-id>"
      wip.app.oidc.client_secret: "<secret>"
    env_file: .env
    environment:
      WIP_BASE_URL: http://wip-caddy:8080
      WIP_API_KEY: ${API_KEY}
      APP_BASE_PATH: /apps/<slug>
      PORT: "<port>"
      NODE_ENV: production
    networks:
      - wip-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "wget", "-qO-", "http://localhost:<port>/apps/<slug>/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 10s

networks:
  wip-network:
    external: true
```

### What `setup-wip.sh` does with this file

1. Reads the `wip.app.*` labels
2. Generates a Caddy reverse proxy route: `/apps/<slug>/*` → `wip-<name>:<port>`
3. If `wip.app.oidc=true`: generates a Dex OIDC client with redirect URIs for both co-located (`https://<hostname>:8443/apps/<slug>/auth/callback`) and remote (`http://localhost:<port>/auth/callback`) mode
4. Generates `start-wip.sh` which prompts the installer to approve which apps to start

## Critical Rules

### 1. No prefix stripping

Caddy forwards the **full path** to the app. If your app is at `/apps/rc`, requests arrive as `/apps/rc/health`, `/apps/rc/auth/callback`, `/apps/rc/wip/api/...`, etc.

Your app **must** mount all routes under `APP_BASE_PATH`:

```typescript
const BASE_PATH = process.env.APP_BASE_PATH || ''
const router = express.Router()

router.get('/health', ...)
router.get('/auth/callback', ...)
router.use('/wip', wipProxy(...))

app.use(BASE_PATH, router)
```

Do NOT rely on Caddy stripping the prefix. Do NOT hardcode paths.

### 2. Use HTTP for internal WIP communication

`WIP_BASE_URL` must be `http://wip-caddy:8080` (HTTP), NOT `https://wip-caddy:8443` (HTTPS).

Caddy has an internal HTTP listener on port 8080 for container-to-container traffic. The HTTPS listener (8443) uses a self-signed cert that Node's `fetch()` / `undici` rejects even with `NODE_TLS_REJECT_UNAUTHORIZED=0`.

### 3. Build-time vs runtime variables

| Variable | When | How |
|----------|------|-----|
| `VITE_BASE_PATH` | Build time | `--build-arg` in Dockerfile |
| `APP_BASE_PATH` | Runtime | Compose environment |
| `WIP_BASE_URL` | Runtime | Compose environment |
| `WIP_API_KEY` | Runtime | From `.env` via `env_file` |
| `OIDC_ISSUER` | Runtime | Compose environment |

Your Dockerfile must declare build-time args:

```dockerfile
ARG VITE_BASE_PATH=/
ENV VITE_BASE_PATH=${VITE_BASE_PATH}
RUN npm run build
```

Build with: `podman build --build-arg VITE_BASE_PATH=/apps/<slug>/ -t <image> .`

### 4. Always include `env_file: .env`

Without it, compose `${VAR}` references are not resolved — they pass through as literal strings like `${API_KEY}`. The `.env` file is generated by `setup-wip.sh` and contains credentials, hostname, and OIDC settings.

### 5. Session cookies behind Caddy

If your app uses sessions (for OIDC):

- Set `app.set('trust proxy', 1)` — Caddy is a reverse proxy, without this Express sees HTTP not HTTPS and `Secure` cookies won't be sent back
- Cookie path = `APP_BASE_PATH` with trailing slash (e.g., `/apps/rc/`)
- Use a unique cookie name (e.g., `rc.sid`) to avoid collisions with other apps on the same host

### 6. OIDC integration

If your app does OIDC with Dex:

- Set `wip.app.oidc: "true"` in compose labels — `setup-wip.sh` generates the Dex client
- `OIDC_ISSUER` = `https://<hostname>:8443/dex` (the external URL, not internal)
- Callback URL = `https://<hostname>:8443/apps/<slug>/auth/callback`
- Include a `state` parameter in the OIDC auth request — Dex returns it and `openid-client` / `oauth4webapi` will reject the response without it
- Use the library's built-in `authorizationUrl()` if possible — don't construct the URL manually

### 7. Disabling an app

Rename the compose chunk: `mv docker-compose.app.<name>.yml docker-compose.app.<name>.yml.disabled`

It won't appear in `start-wip.sh`'s app list.

### 8. Remote mode (app on Mac, WIP on Pi)

An app can run on a different machine from WIP. Override `WIP_BASE_URL` in `.env`:

```
WIP_BASE_URL=https://pi-poe-8gb.local:8443
```

The app container runs standalone (just its own compose chunk), not with the full production compose. The Dex client includes `http://localhost:<port>/auth/callback` for this mode.

## Building and Pushing an App Image

### Node/React apps (typical)

```bash
# Pack WIP client libraries (if not already in libs/)
cd /path/to/wip-repo/libs/wip-client && npm pack --pack-destination /path/to/app/.docker-libs/
cd /path/to/wip-repo/libs/wip-react && npm pack --pack-destination /path/to/app/.docker-libs/
cd /path/to/wip-repo/libs/wip-proxy && npm pack --pack-destination /path/to/app/.docker-libs/

# Build
cd /path/to/app
podman build --build-arg VITE_BASE_PATH=/apps/<slug>/ -t gitea.local:3000/peter/<name>:<tag> .

# Push
podman push --tls-verify=false gitea.local:3000/peter/<name>:<tag>

# Clean up
rm -rf .docker-libs/
```

### Dockerfile pattern

Multi-stage build: Node for build, Node (or nginx) for runtime.

```dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY package.json package-lock.json ./
COPY libs/ libs/           # or .docker-libs/ for packed tarballs
RUN npm ci --ignore-scripts
COPY . .
ARG VITE_BASE_PATH=/
ENV VITE_BASE_PATH=${VITE_BASE_PATH}
RUN npm run build

FROM node:20-alpine AS production
WORKDIR /app
COPY package.json package-lock.json ./
COPY libs/ libs/
RUN npm ci --omit=dev --ignore-scripts && npm cache clean --force
COPY server/ server/
COPY tsconfig.json ./
COPY --from=build /app/dist dist/
ENV NODE_ENV=production
EXPOSE 3011
HEALTHCHECK --interval=30s --timeout=5s --start-period=10s \
  CMD wget -qO- http://localhost:3011/${APP_BASE_PATH:-}/health || exit 1
CMD ["npx", "tsx", "server/index.ts"]
```

## Deployment Checklist

- [ ] Compose chunk has all required `wip.app.*` labels
- [ ] `env_file: .env` is present
- [ ] `WIP_BASE_URL` is `http://wip-caddy:8080` (not HTTPS)
- [ ] `APP_BASE_PATH` matches the `wip.app.route` label
- [ ] All Express routes mounted under `APP_BASE_PATH`
- [ ] Dockerfile has `ARG VITE_BASE_PATH` before `npm run build`
- [ ] Image built with `--build-arg VITE_BASE_PATH=/apps/<slug>/`
- [ ] Image pushed to registry
- [ ] `setup-wip.sh` re-run after adding the compose chunk
- [ ] If OIDC: `trust proxy` set, cookie path has trailing slash, unique cookie name, state parameter in auth request
- [ ] Health check URL includes the base path

## Common Failures

| Symptom | Cause | Fix |
|---------|-------|-----|
| 502 on API calls | TLS error proxying to `https://wip-caddy:8443` | Use `http://wip-caddy:8080` |
| 401 on API calls | `WIP_API_KEY` mismatch | Full restart so all containers read same `.env` |
| 404 on static assets | `VITE_BASE_PATH` not baked at build time | Add `ARG`/`ENV` to Dockerfile, rebuild with `--build-arg` |
| OIDC "unexpected state" | State parameter not sent in auth request | Use `client.randomState()`, store in session, pass to callback |
| OIDC redirect to wrong app | Callback URL missing base path | Mount auth routes under `APP_BASE_PATH`, don't strip prefix in Caddy |
| Session lost after Dex redirect | `trust proxy` not set or cookie path wrong | `app.set('trust proxy', 1)`, cookie path = `BASE_PATH/` with trailing slash |
| `${VAR}` literal in container | Missing `env_file: .env` | Add `env_file: .env` to compose chunk |
| PostgreSQL auth failed | `.env` regenerated, volume has old password | Never delete `.env` without wiping PostgreSQL volume |
| Dex "no password hash" | OIDC client appended to wrong YAML section | Ensure `setup-wip.sh` inserts before `connectors:`, not at EOF |
